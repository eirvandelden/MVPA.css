Welcome to MVPA.css

This page demonstrates all components and features of the MVPA.css framework. The framework uses semantic HTML5 elements and OKLCH colors with automatic dark mode support.

Theme Switcher: Use the buttons below to test different themes and color schemes. Your preference will be saved.

How to test themes

You can change the theme in two ways:

  1. Use the theme switcher buttons above (these persist in localStorage)
  2. Open your browser's developer tools (right-click → Inspect) and change the data-theme attribute on the <html> element to test other themes

Available themes: selenized_light, selenized_dark, white, black

Available color schemes: system, light, dark

Details & Summary (Disclosure Widgets)

Installation Instructions

Import all 18 CSS files in the correct order, then set the data-theme attribute on the html element.

<html data-color-scheme="system" data-theme="selenized_light">
  <!-- Your content -->
</html>
Theme System (Open by Default)

The framework supports four themes with automatic dark mode switching. Use the data-theme attribute to override.

  • Selenized Light (warm, default light theme)
  • Selenized Dark (dark variant with warm accents)
  • Pure White (clean minimal)
  • Pure Black (maximalist)
Color Palette

The framework uses OKLCH color space with Selenized colors. Each theme defines background and foreground shades plus semantic colors (success, danger, warning, info).

Background Shades
5 levels: lightest, lighter, base, darker, darkest
Foreground Shades
5 levels: darkest, darker, base, lighter, lightest
Semantic Colors
Primary (blue), Success (green), Danger (red), Warning (yellow), Info (cyan)
Design Tokens

All design values are exposed as CSS custom properties for easy customization and theming.

  • Color variables (OKLCH based)
  • Spacing system (inline/block in 5 levels)
  • Typography scale (8 sizes)
  • Line heights, border radius, transitions
  • Z-index stack, max widths

What makes animations delightful?

Great animations respond to user intent, are subtle enough not to distract, and make interactions feel intentional. They should never delay functionality – the action happens immediately, the animation is just the cherry on top.

How do these animations perform?

All animations in MVPA.css only animate transform and opacity, which are GPU-accelerated for smooth 60fps performance. This ensures the animations delight without slowing down your app.

Accessibility & Reduced Motion

All animations respect the prefers-reduced-motion preference. Users who prefer reduced motion will see interactions without animations, but functionality remains completely intact.

To test on macOS: System Settings → Accessibility → Display → Reduce motion

Turbo Integration

When using this with Turbo Streams and Stimulus, elements added via Turbo automatically get the appropriate animations:

  • data-turbo-stream-transition="slide-in" – Slides in from right
  • data-turbo-stream-transition="highlight" – Highlights with a flash
  • data-turbo-stream-transition="fade-out" – Fades out before removal
